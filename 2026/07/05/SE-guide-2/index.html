BUPT 软工生存指北-大二篇

发表于2026-07-05|更新于2026-07-10|study

大二上

  有一说一，大二上是真的很闲，没有早八、一周四休的生活还能回来吗。前提是只选修最低限度的课。

数据结构 by 张

  408 科目之一，网上有很多教学资源，无论是做题还是可视化教学，但要注意和学校教学内容的小差别。有 7 次实验，基本是发个文档自己写代码，也不需要验收，只用提交代码和报告，有 AI 帮助非常简单。期中的题目我有记下来，未来会整理提交到 BYR Docs 上。期末来不及记题了，但比预想的简单，基本是作业题的难度，也不会考手写各种树，但也有不少代码题。最后一道大题是手写希尔排序的代码。

计算机组织与结构 by 熊 & 夏

  408 科目之一，但多了一点体系结构的内容。上课一般是纸笔签到，有点名提问，而且有课堂练习，需要手写提交。有 20% 的实验部分，使用 TEC-PLUS 实验箱完成 6 个实验，现场验收，按照验收先后次序给分，越早完成得分越高，并且手写实验报告，于期末当天提交。实验部分比较折磨，因为硬件到底还是要上手实操，接线按键之类的都需要手动来，没法让 AI 代劳，而且存在设备故障的可能。关于期末，我做过一个节选了部分章节的计组笔记，其中附录部分包含了 24 级的期中和期末试题。最后一节课非常重要，老师会划简答题的出题范围。

汇编语言设计实践

  实践课，有 6 个实验，其中最后一个要现场验收，前五个只用写报告。开发环境十分复古，最后写报告还需要各种截图，实在做不明白就找找祖传代码吧。最后一个实验是用汇编实现一个电子琴，根据键盘按键发出不同频率的音。我验收的时候问我的问题大意是：按下 A 键不松再按 S 键，程序继续发 A 的音而不会发 S 的，结合代码逻辑解释其原因。我其实也不知道是为什么，只好糊弄过去，好在没有过多拷打。最后给分也挺高。

商务英语（选修）by 张

  避雷。事多，其中 pre 包括商务演讲、模拟面试、商务谈判，商务演讲是一个融资路演，需要进行市场分析并介绍你的产品和公司团队；模拟面试需要自己准备简历，既需要面试别人也需要被别人面试；商务谈判需要同时准备校方代表和学生代表的内容，谈判当天由抽签决定谈判对象。以上 pre 内容均需要组队完成。有几次随堂小测，内容为课内知识点，无法靠底力完成。最阴的是最后一周的闭卷期末，有听力有作文，考的也都是课内知识点，令人无语。

Linux 环境及开发工具应用实践（选修）

  Linux 和基于 C++ 的编程设计二选一，后者我没选，内容似乎是做一个 C++ 项目。Linux 实践包含三个实验，一是装系统以及编写 Shell 脚本，二是做一个 Shell 项目并展示，三是在 Linux 系统下做一个 C/C++ 项目并展示。第一个实验是个人完成，第二、三是小组完成，展示验收时有现场提问。体验中规中矩。

大二下

  大二下有个形式语言与自动机，虽然我没选但听说体验极差，尽管是“考查”但还是有闭卷期末。开课老师更是“好评如潮”，关键词“球”。

数据库系统原理 by 孙

  有 6 个实验，环境需要需要在华为云 ECS 服务器上部署 openGauss 数据库，这个跟着文档做就行，老师会提供服务器代金券，此外实验四还需要安装 Power Desinger 用于数据库设计。实验内容包括 SQL 语句练习、数据库的备份与恢复等，均需要在服务器上完成。不过这些实验不需要验收，只需要提交报告，记得截图。关于期中期末，我和几位同学共同整理了本学期的期末试题，加上我自己记录的期中试题，一并上传到了 BYR Docs 上，在 wiki 页面标有“（软件学院）”。

操作系统原理 by 赵

  408 科目之一，考核包括展示、实验和期中期末。展示需要组队完成，选择一个主题进行调研，老师会提供一个主题参考，按照主题难度等级有不同给分，比如 OS 原理属于 85 分档，结合大模型的就是 90 分以上，每周有一两个小组展示，需要做 PPT、写报告。实验一共有 6 个，同样需要组队完成，且需要现场验收，内容基本是用代码实现 OS 的部分功能，比如进程通讯、调度算法、替换算法等。虽然一个小组只需要一个人验收，但每个人都要提交报告。期中期末的简答题都来自教材习题，而且 PV 大题有很大可能考一次课上的习题，我这届是期中考的。我和几位同学共同整理了本学期的期末试题并传到了 BYR Docs 上，其中注明了简答题和部分大题对应的教材习题。
  顺带一提，可以选择自己做个操作系统，以小组为单位，要求至少包含进程管理、内存管理及文件系统等主要内容，并向全班展示，完成者可以得到期中期末以外成绩的满分。

概率论与数理统计（3 学分）

  概率论的题型较为固定，有相对较多的往年题可供参考，也有大量的网课资源用于学习。期中是一个大作业，需要每人提交一份论文，也可以选择参加统计建模竞赛，用参赛论文作为期中论文。有两次线上小测。如果高数基础不好的话，最好先复习一下微积分知识。

数据结构课程设计（选修）by 李

  二选一的 1 学分课设。当时内容就一句话：“设计一款展现中国精神的自定义应用软件”，形式可以是网页、小程序、游戏之类，最多 5 人一组完成，最后一周汇报，并提交小组和个人实验报告。需要注意的是要体现数据结构课程的内容，比如队列、堆栈、图等。有 AI 的帮助，做起来还是比较轻松的。

网球（选修）by 崔

  因为各种原因（劳动节、校运会、端午、恶劣天气），一学期的网球课只有一半是在线下上的，另一半就放假或者转线上腾讯会议了。不签到，运动强度不高，主要练习颠球、挥拍、正反手击球。由于线下课较少，反手部分只在最后一节有简单的练习，期末考核只有正手发球。这也极大降低了容错：发球部分占了总成绩 40%，一共发 5 个球，也就是一个球就占了期末 8 分，如果出界或没过网就是 0 分，过网但没过对面发球线是 4 分，每人仅有一次机会，不能重来。比起技术更多考验心态。还有 30% 是发球动作的评分。虽然考发球的时候软脚了，但似乎其他部分都给了满分。

软件产品综合研发实践（阶段一）

  只有 6 周的实验课，一共两个实验：一是在 Github 或 Gitee 提交一次 PR，要求有 10 行以上的代码改动，以修复一个 issue 或实现一个新功能，并且必须使用 AI 辅助完成；二是使用 vibe coding 工具做一个 agent 项目，要求 agent 至少有两种不同的功能，使用 MCP 或者函数调用，代码必须由 AI 编写。两个实验都没有验收，都需要提交报告，写上仓库地址并附截图。
